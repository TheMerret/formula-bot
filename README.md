# 🤖 Formula Recognition Telegram Bot

Telegram-бот для распознавания математических формул с использованием GigaChat API. Бот анализирует изображения формул и предоставляет подробные объяснения.

## 📋 Описание

Бот реализует следующую функциональность:
- **Распознавание формул**: Загрузка изображения формулы (написанной от руки или из учебника)
- **Трехэтапная обработка** (LangChain chains):
  1. Извлечение текста формулы из изображения
  2. Поиск дополнительной информации в интернете (DuckDuckGo)
  3. Получение подробного объяснения формулы с контекстом
- **Интеграция с GigaChat API**: Использование LLM для анализа и объяснения
- **Интернет-поиск**: Автоматический поиск информации о формулах через DuckDuckGo
- **Кэширование**: SQLite кэш для ускорения повторных запросов
- **Регистрация пользователей**: SQLite база данных для хранения информации о пользователях
- **FSM (Finite State Machine)**: Управление состояниями диалога

## 🚀 Установка и запуск

### Вариант 1: Запуск с Docker Compose (рекомендуется)

#### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd ainj
```

#### 2. Настройка переменных окружения

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Заполните `.env` своими данными:

```env
BOT_TOKEN=your_telegram_bot_token_here
GIGACHAT_KEY=your_gigachat_credentials_here
GIGACHAT_SCOPE=GIGACHAT_API_PERS
GIGACHAT_MODEL=GigaChat-Pro
DATABASE_PATH=users.db
LOG_LEVEL=INFO

# Search Configuration
ENABLE_SEARCH=true
SEARCH_MAX_RESULTS=5
SEARCH_CACHE_TTL=86400
```

#### Получение токенов:

**Telegram Bot Token:**
1. Найдите [@BotFather](https://t.me/botfather) в Telegram
2. Отправьте `/newbot`
3. Следуйте инструкциям
4. Скопируйте полученный токен

**GigaChat API Key:**
1. Зарегистрируйтесь на [developers.sber.ru](https://developers.sber.ru/)
2. Создайте проект
3. Получите credentials для GigaChat API

#### 3. Запуск с Docker Compose

```bash
# Создать и запустить контейнер
docker-compose up -d

# Просмотр логов
docker-compose logs -f

# Остановить контейнер
docker-compose down

# Перезапустить контейнер
docker-compose restart
```

#### 4. Управление контейнером

```bash
# Просмотр статуса
docker-compose ps

# Остановить и удалить контейнер с данными
docker-compose down -v

# Пересобрать образ
docker-compose build --no-cache

# Обновить и перезапустить
docker-compose up -d --build
```

## 📱 Использование

### Команды бота

- `/start` - Регистрация и главное меню
- `/help` - Справка по использованию
- `/formula` - Начать распознавание формулы
- `/cancel` - Отменить текущее действие

### Кнопки меню

- **📸 Распознать формулу** - Войти в режим распознавания
- **ℹ️ Помощь** - Показать справку
- **❌ Отмена** - Вернуться в главное меню

### Процесс работы

1. Запустите бота командой `/start`
2. Нажмите "📸 Распознать формулу" или используйте `/formula`
3. Отправьте фото формулы
4. Дождитесь анализа (25-35 секунд с поиском, 20-25 секунд при кэше)
5. Получите подробное объяснение с источниками

### Примеры формул

Бот может распознать и объяснить:
- **E = mc²** - Формула Эйнштейна
- **F = ma** - Второй закон Ньютона
- **a² + b² = c²** - Теорема Пифагора
- **V = πr²h** - Объем цилиндра
- И многие другие математические и физические формулы

```sql
CREATE TABLE users (

    username TEXT,                    -- Telegram username
    first_name TEXT,                  -- Имя пользователя
    last_name TEXT,                   -- Фамилия пользователя

```

**Таблица search_cache:**
```sql
CREATE TABLE search_cache (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    formula_normalized TEXT UNIQUE,      -- Нормализованная формула
    search_query TEXT,                   -- Поисковый запрос
    results TEXT,                        -- Результаты поиска (JSON)
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    expires_at TIMESTAMP                 -- Время истечения кэша
)
```

## 🔄 Цепочки обработки (LangChain)

### Chain 1: Извлечение формулы из изображения
```
Изображение → GigaChat Vision → Текст формулы (LaTeX)
```
**Время**: ~10-15 секунд

### Chain 2: Поиск в интернете (NEW! 🆕)
```
Текст формулы → DuckDuckGo Search → Контекстная информация
```
**Время**: 2-3 секунды (или <1 секунды при кэше)

**Особенности:**
- Автоматический поиск информации о формуле
- Кэширование результатов на 24 часа
- Повторные запросы выполняются мгновенно
- Можно отключить через `ENABLE_SEARCH=false`

### Chain 3: Объяснение формулы с контекстом
```
Текст формулы + Результаты поиска → GigaChat + Prompt Template → Подробное объяснение
```
**Время**: ~10-15 секунд

Объяснение включает:
1. Название формулы
2. Область применения
3. Описание явления
4. Расшифровка переменных
5. Практическое применение
6. Пример использования
7. **Источники информации** (🌐 DuckDuckGo + 🤖 GigaChat)

**Общее время обработки:**
- С поиском (первый раз): ~25-35 секунд
- С кэшем: ~20-25 секунд
- Без поиска: ~20-25 секунд

## 🔍 Интернет-поиск (NEW! 🆕)

### Конфигурация

Настройки поиска в `.env`:

```env
# Включить/выключить поиск
ENABLE_SEARCH=true

# Максимальное количество результатов поиска
SEARCH_MAX_RESULTS=5

# Время жизни кэша в секундах (по умолчанию 24 часа)
SEARCH_CACHE_TTL=86400
```

### Как это работает

1. **Автоматический поиск**: После извлечения формулы бот автоматически ищет информацию в интернете
2. **Кэширование**: Результаты сохраняются в базе данных на 24 часа
3. **Быстрые повторные запросы**: Повторные запросы той же формулы выполняются мгновенно
4. **Источники**: Результаты поиска добавляются к объяснению GigaChat

### Преимущества

- ✅ Более полная информация о формулах
- ✅ Актуальные данные из интернета
- ✅ Ссылки на дополнительные ресурсы
- ✅ Кэширование для быстрых повторных запросов
- ✅ Можно отключить при необходимости

### Отключение поиска

Если нужно отключить интернет-поиск:

```env
ENABLE_SEARCH=false
```

Бот продолжит работать, используя только GigaChat для объяснений.
